Protocolo HTTP

Protocolo de Transferência de Hipertexto é um dos protocolos web mais importantes, utilizado para acessar páginas web. Os serviços que implementam tal protocolo se comunicam através das portas 80 para HTTP e 443 para HTTPS (Porta criptografada).


Verbos HTTP

Métodos utilizados para indicar ação desejada a ser realizada no recurso identificado.

  • GET: Buscar recursos e Cache;
  • POST: Criar novo recurso;
  • PUT/PATCH: Atualizar recurso existente;
  • DELETE: Remover recurso.

Códigos de status de respostas HTTP

Indicam, em feedback, se requisição HTTP foi corretamente concluída, sendo agrupadas em 5 classes:

  1. Respostas informativas (100-199);
  2. Respostas de sucesso (200-299);
  3. Redirecionamentos (300-399);
  4. Erros do cliente (400-499);
  5. Erros do servidor (500-599).

Respostas informativas
  • 100 Continue: Resposta provisória indicando conformidades até o momento, onde o cliente pode continuar com a requisição ou ignorá-la;
  • 101 Switching Protocol: Código enviado em resposta de header de solicitação Upgrade pelo cliente, indicando protocolo a que o servidor está alternando;
  • 102 Processing: Voltado à WebDAV, indica que servidor recebeu e está processando a requisição, mas nenhuma resposta encontra-se disponível ainda;
  • 103 Early Hints: Tem objetivo de uso como header link, indicando que o agente deve iniciar a pré-carregar recursos enquanto servidor prepara resposta;

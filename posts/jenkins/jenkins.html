Conceito

Sistema de automação de tarefas, desenvolvido em Java, utilizado para CI/CD, com inúmeros plugins e integrações disponíveis. Criado via arquivo WAR ou executado via webserver, como Tomcat. Possui GUI web e via APIs de conexão, além de suporte a pipelines e arquivo declarativo (Jenkinsfile). Abaixo, principais funcionalidades:

  • Realizar checkout de aplicação em repositório logo após commit, juntamente build da mesma;
  • Executar processos de CI (build, test, performance, tarefas necessárias para verificação do funcionamento da aplicação);
  • Gerar relatório (output) dos passos (jobs) executados;
  • Realizar deploy automatizado;
  • CRON (tasks automatizadas) com muitas funcionalidades, permitindo criar jobs automatizados por tempo de execução, independente de pipeline.

Instalação

Instalação do Jenkins via Docker (necessário docker):

  • Comando Docker container Jenkins: "docker run -u root -d --name jenkins -p 8080:8080 -p 50000:50000 -v "C:\Users\NOME_USUARIO\Desktop\jenkins_home:/var/jenkins_home" jenkins/jenkins"
  • (opcional) Localizar senha 1º acesso: "docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword"
  • Acessar Jenkins GUI no navegador: http://localhost:8080
  • No Jenkins GUI, informar senha 1º acesso, selecionar Instalar extensões sugeridas
  • Informe credeciais de preferência (exemplo usuário 'ubsocial', senha '2024')

    • Outros comandos:
  • Acessar container Jenkins: "docker exec -it jenkins bash"
  • (opcional) Ver logs do container: "docker logs jenkins"

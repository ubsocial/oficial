Ansible

Automatização simplificada de SOs

Resumo em construção
Voltar

Conceito:

Ferramenta Agentless com objetivo de automatizar a gestão (Instalações, configurações, provisionamento, etc) em diferentes SOs servidores simultaneamente (Linux, Windows, etc). Entre vários nodes (Nós-Máquinas), tem-se o Master node (Controlador), onde são informados os demais nodes e respectivas credenciais. Para isso, recomenda-se SSH ativo nos demais nodes (Workers ou Slaves - Controlados). O Ansible possui vários Módulos, que exercem funções específicas.


Instalação (Debian):

  • Pré-requisito: sudo apt install python3-pip
  • Instalação: pip3 install ansible
    • Verificar: ansible --version
  • Após instalado, gerará em '/etc/ansible':
    • ansible.cfg: Arquivo de configurações
    • hosts: Arquivo de inventário, onde serão declarados os nodes
    • roles: ''''''''
  • VSCodes extensão oficial Ansible

Lab Cloud:

  • Mater node - CentOS 7
  • Worker node1 - Debian 11
  • Worker node2 - Ubuntu Server 22.04 LTS

Configuração:

Na máquina local, acessar Master node via SSH (Supondo que há chave de autenticação cadastrada no Master node): 'ssh nomeUsuario@nomeMasterNode'.


Em 'hosts', informar:
[distros]
172.104.18.126 # IP do Debian, Worker 1
172.104.18.156 # IP do Ubuntu, Worker 2

[distros:vars]
ansible_user=root # User e Password dos Workers. O ideal é criar usuário específico para tal, e utilizar chave de autenticação ao invés de senha simples. Como fora dessa forma, deve-se autorizar no 'ansible.cfg'
ansible_password=123

Em 'ansible.cfg', descomentar linha:
host_key_checking = False

Testar conexão geral:
ansible distros -m ping

Dicas

Para executar comandos em diferentes distros, o ideal é criar grupos de distros similares (Ex: Distros Debian, Distros Arch), ou arquivos de inventário diferentes. Após, executar o comando específico (Ex: sudo apt update) e direcioná-lo para o grupo de distros alvo. Exemplo:


ansible distros -a "sudo apt update"
ansible distros -a "uptime"

Playbook

Arquivo YAML (playbook.yml) com informações para a gestão automatizada do Ansible nos nodes. Após criado o arquivo, executa-se o comando 'ansible-playbook playbook.yml' para criar a gestão. Exemplo de sintaxe, instalar nginx nos workers (Após executado, pode-se testar com 'curl ipWorker' e mostrará o código nginx):


---
- name: Instalar o nginx
  hosts: distros
  become: yes
  tasks:
  - name: Instalar servidor web
     package:
      name: nginx
      state: present # present: Instalado, absent: Não deseja no servidor, latest: Instalar última versão

Vagrant

Gerenciamento automatizado de VM

Resumo em construção
Voltar

Introdução

Solução de gerenciamento automatizado e construção de laboratórios de máquinas virtuais (VMs), via arquivo único ('Vagrantfile'), descartando todas etapas de interações com SO e suas necessidades, conectado a um Provider. Pode ser conectado localmente ou até mesmo em Clouds (AWS, Azure, GCP, Digital Ocean, etc). Para utilizar o Vagrant, é necessário possuir VirtualBox. Códigos conforme documentação oficial: Site oficial.

Conceitos

  • Box: SO definido no início do projeto, que terá sua imagem baixada pelo Vagrant na VM. O VagrantUp é o repositório oficial dos Boxes: Acesse;
  • Host e Guest: Host é o sistema que está executando o Vagrant (SO da máquina local). Guest é o SO da VM;
  • Provider: Hypervisor responsável pela virtualização (VirtualBox-default, VMWare, KVM, Docker, Hyper-V);
  • Plugins: Extensões do Vagrant, que podem ser baixadas, como conexão Clod;
  • Provisioner: Software orquestrador que irá automatizar as tarefas executadas na VM (Chef, Puppet, Ansible, Salt, arquivos, Shell Script manualmente, etc). Consiste em scripts e arquivos com códigos de automação do processo de criação da VM;
  • Synced Folders: Diretórios usados para compartilhar conteúdo entre Host e Guest;
  • Vagrantfile: Arquivo de configuração do Vagrant, onde são definidas configurações da VM, versão do SO, memória, ip, etc;

Comandos básicos


Instalação (Debian): sudo apt install vagrant
vagrant init: Gerar novo Vagrantfile baseado em uma Box
vagrant up: Iniciar VM e executadar Provisioner (Subir ambiente)
vagrant reload: Reiniciar VM
vagrant provision: Executar Provisioner
vagrant halt: Desligar VM (destroy excluir VM)
vagrant suspend: Pausar VM (resume retomar VM)
vagrant ssh: Acessar ssh da VM
vagrant box list: Listar Boxes
vagrant box remove hashicorp/bionic64: Remover Box específico #No caso, removeria o hashicorp/bionic64

Vagrantfile

Estrutura do arquivo Vagrantfile e exemplo passo a passo de sua construção:


1.Estrutura básica:
Vagrant.configure("2") do |config|
    Conteúdo aqui
end

2.Boxes: (Executar vagrant up para criar, depois vagrant ssh para acessar)
Vagrant.configure("2") do |config|
    Conteúdo aqui, exemplos abaixo:
    config.vm.box = "ubuntu/xenial64" #Definir Box (SO e versão opcional)
end

3.Provisioner: (Executar vagrant provision para atualizar)
Vagrant.configure("2") do |config|
    config.vm.box = "ubuntu/xenial64"
    config.vm.provision "shell",
    inline: "apt update"
    config.vm.provision "shell", inline:<<-SHELL
        apt install -y apache2
    SHELL
end

4.Especifiar porta de acesso: (Executar vagrant reload para reiniciar. Após, acessar porta 8080 da VM o apache2)
Vagrant.configure("2") do |config|
    config.vm.box = "ubuntu/xenial64"
    config.vm.network "forwarded_port", guest: 80, host: 8080, host_ip: "127.0.0.1"
    config.vm.provision "shell",
    inline: "apt update"
    config.vm.provision "shell", inline:<<-SHELL
        apt install -y apache2
    SHELL
end

Synced Folders

O diretório do Vagrantfile (Host) e o diretório '/vagrant' (Guest) são sincronizados por padrão. Para testar, crie, na VM, algo dentro do '/vagrant', saia da VM e vá ao diretório do Vagrantfile do Host para verificar se sua criação também encontra-se lá. Para modificar os padrões, segue linha de código no Vagrantfile:


Vagrant.configure("2") do |config|
    config.vm.synced_folder "src/", "/srv/website" #Diretório Host, diretório Guest
end

Exemplo prático

Através do Vagrant, criar VM Ubuntu com servidor web nginx e acessá-lo com o Host. No caso, o Vagrantfile será criado manualmente no Host, dispensando 'vagrant init'.


1.Criar diretório do projeto e Vagrantfile: mkdir testeVagrant && cd testeVagrant && touch Vagrantfile

2.Inserir conteúdo abaixo dentro do Vagrantfile:
Vagrant.configure("2") do |config|
    config.vm.box = "ubuntu/xenial64"
    config.vm.network "forwarded_port", guest: 80, host: 85
    config.vm.provision "shell",
    inline: "apt update"
    config.vm.provision "shell", inline:<<-SHELL
        apt install -y nginx
        service nginx start
        touch /var/www/html/ubsocial.html
        echo "ubsocial nginx funcionando" > /var/www/html/ubsocial.html
    SHELL
end

3.Criar Box: vagrant up

4.Acessar nginx via Host: Na máquina local (Host), acessar no browser url 'localhost:85/ubsocial.html'
